# Lumen Analytics

A fictional B2B SaaS customer behavior and revenue analytics platform,
built as a portfolio demo for Paradigm Coding Solutions.

**Live:** https://lumenanalytics.projectnexuscode.org (pending Phase 0 deploy)

> See changes in your business before they hit your P&L.

Lumen showcases data-dense interface design and applied ML: automated
anomaly detection on revenue and engagement metrics, with explanatory
drill-down that ranks which customer segments drove each anomaly.

All customers, metrics, and events are synthetic, generated by a
deterministic seeded pipeline so the demo is reproducible run to run.

## Stack

- Next.js 14 (App Router), TypeScript, Tailwind, shadcn/ui
- Recharts (time series), D3 (cohort heatmap, Sankey funnel)
- SQLite via better-sqlite3, populated by a synthetic data generator
- Anomaly detection: STL-style trend + weekly seasonality decomposition,
  flagging |residual| > 2.5 sigma
- Cause attribution: per-slice anomaly scoring across plan tier,
  geography, and industry, ranked by contribution

## Develop

```powershell
npm install
npm run seed     # builds data/lumen.db (deterministic, ~80k rows)
npm run dev      # http://localhost:3000
```

## Data model

`data/lumen.db` is a build artifact (gitignored). Tables: `users`,
`customers`, `events`, `metrics_daily`, `anomalies`,
`customer_mrr_monthly`, `customer_usage_daily`. The metric catalog lives
in `src/lib/data/catalog.ts`.

Sliced metrics are generated at the (plan tier x geography x industry)
cell level and aggregated, so every dimension's slices sum to the same
top line and the three scripted anomalies (churn spike, expansion surge,
feature adoption drop) are genuinely concentrated in specific segments
for the attribution engine to find.

## Verification

The `verify/` directory declares what "passing" means for this repo.
See `verify/README.md` for the full convention.

**Quick smoke (every PR):**

```powershell
# From the verify skill or CI:
bash verify/ci/quick_smoke.sh verify/smoke.yml
```

Hits all surfaces in `verify/smoke.yml` -- home, app redirect, anomaly
log redirect, anomaly API, team API, and session API. Curls only; no
browser required. Skips `selector_present` and `axe_no_critical` (those
need a browser and run locally).

**Deep verify (tier-3 PRs, before merge):**

```powershell
# Run the full /verify deep pass locally with the verify skill,
# then commit the report to verify/reports/ with "Overall: PASS".
# CI enforces the report exists before allowing merge.
bash verify/ci/deep_gate.sh
```

Tier-3 surfaces that require deep verify before merge:

- `anomaly-detection-backend` -- the STL decomposition + episode detector
  that generates all anomaly data (src/lib/ml/, scripts/detect.ts)
- `anomaly-log` -- /app/anomalies, the primary anomaly story entry point
- `anomaly-detail-panel` -- /api/anomalies/[id] + triage status endpoint
- `demo-auth` -- /api/session, middleware, /api/portal/handoff

**CI workflow:** `.github/workflows/verify.yml` runs quick-verify on every
PR and deep-verify (report gate) on any PR labeled `tier-3`.

**Surface assertions:** `verify/assertions/` holds the full deep-verify
assertion sets per surface (home, anomaly-log, anomaly-detail).
`verify/smoke.yml` holds the fast subset that runs on every PR.
